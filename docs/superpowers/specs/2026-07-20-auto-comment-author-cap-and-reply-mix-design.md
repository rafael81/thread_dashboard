# 자동댓글 계정 제한 및 내 글 대댓글 혼합 설계

## 목적

자동댓글이 동일 계정의 여러 게시글에 짧은 시간 동안 집중되는 현상을 막고, 외부 게시글 자동댓글과 내 게시글에 달린 댓글의 자동 대댓글을 예측 가능한 비율로 함께 처리한다.

## 확정 정책

- 상호 팔로우 여부는 검사하지 않는다.
- 일반 자동댓글은 대상 계정별로 KST 날짜 기준 하루 최대 2건만 게시한다.
- 내 글 대댓글은 내 원글별로 날짜와 무관하게 누적 최대 2건만 게시한다.
- 일반 자동댓글 5건이 성공할 때마다 처리 가능한 내 글 대댓글 1건을 우선 처리한다.
- 처리 가능한 내 글 대댓글이 없으면 일반 자동댓글은 대기하지 않고 계속 게시한다. 이때 5건 카운터는 유지하여 대댓글이 발견되는 즉시 우선 처리한다.
- 게시에 성공한 항목만 5건 카운터에 반영한다. 검수 거절, 제한으로 인한 제외, 게시 실패는 세지 않는다.
- 내 글 댓글의 하트 자동화는 원글별 대댓글 2건 제한과 무관하게 기존대로 수행한다.
- 두 게시 경로 모두 기존 Grok 문맥 필수, Gemini 검수 및 결정적 품질 게이트를 유지한다.

## 데이터와 판정

### 일반 자동댓글 계정별 일일 제한

대상 계정은 정규화된 X 상태 URL의 handle에서 구한다. KST 오늘의 `commentHistory`에서 같은 handle로 성공 게시된 일반 자동댓글 수를 센다. 같은 날짜에 게시될 가능성이 있는 `pendingCommentPosts`도 예약 수로 합산하여 prefill 단계에서 2건을 초과해 새 초안을 만들지 않는다.

기존 pending에 이미 동일 계정 초안이 2건보다 많이 있다면 삭제하지 않는다. consumer가 게시 직전에 한도를 다시 검사하고, 초과분은 다음 KST 날짜 경계 이후까지 보류한다. 이 보류는 기술 실패나 품질 격리가 아니며 재시도 횟수를 증가시키지 않는다.

### 내 원글별 누적 대댓글 제한

내 원글의 정규화 URL을 키로 사용한다. `ownPostReplyHistory`의 성공 게시 기록과 대댓글 pending/수동 큐의 예약 건수를 합산한다. 합계가 2건이면 해당 원글에서는 새 대댓글 후보를 생성하거나 게시하지 않는다.

제한은 날짜가 바뀌어도 초기화하지 않는다. 기존에 자동화가 게시한 성공 기록을 기준으로 적용하며, 동일 상대 댓글 URL에 대한 기존 중복 방지도 그대로 유지한다.

## 공유 게시 조정

영속 상태에 `successfulAutoCommentsSinceOwnReply` 카운터를 둔다.

1. 일반 자동댓글 게시 성공 시 카운터를 1 증가시킨다.
2. 카운터가 5 미만이면 기존 일반 자동댓글 consumer가 계속 동작한다.
3. 카운터가 5 이상이면 다음 게시 기회에 처리 가능한 내 글 대댓글 한 건을 우선한다.
4. 대댓글 게시 성공 후 카운터를 0으로 초기화한다.
5. 후보가 없거나 대댓글 생성·검수가 실패하면 카운터를 유지하고 일반 자동댓글을 막지 않는다.

내 글 대댓글 작업은 한 번의 기회에 정확히 한 건만 게시한다. 전체 원글 순회를 한 번에 게시하는 기존 장시간 작업을 공유 게시 경로에서 호출하지 않고, 발견·생성 결과 중 한 건만 소비하는 제한된 경로를 사용한다.

일반 자동댓글과 내 글 대댓글은 동일 X writer lock을 통해 직렬화한다. Grok과 Gemini 준비 작업은 각자의 전용 세션·포트를 유지하므로 게시 writer 외에는 서로 경합하지 않는다.

## 오류 처리

- 계정 handle을 URL에서 확정할 수 없는 일반 댓글 후보는 자동 게시하지 않는다.
- KST 날짜 계산 실패 시 한도 검사는 fail-closed로 동작한다.
- 일일 한도에 걸린 pending은 `nextEligibleAt`을 다음 KST 자정으로 설정한다.
- 원글별 누적 2건을 채운 대댓글 pending은 더 이상 게시하지 않되, 삭제 대신 명시적인 `root_post_reply_cap_reached` 상태로 기록하여 대시보드에서 이유를 확인할 수 있게 한다.
- 서버 재시작 시 카운터와 pending 상태를 복원한다.

## 대시보드 관측성

자동화 상태에 다음 정보를 노출한다.

- 일반 자동댓글 계정별 일일 상한 `2`
- 현재 5:1 카운터
- 대댓글 우선 처리 대기 여부
- 한도 때문에 보류된 일반 댓글 수
- 원글별 대댓글 2건 제한으로 제외된 수와 사유

## 테스트 및 완료 조건

- KST 같은 날짜에 동일 계정 일반 댓글이 2건까지만 허용되고 세 번째가 차단된다.
- UTC 날짜가 아니라 KST 날짜 경계에서 일반 댓글 한도가 초기화된다.
- 게시 기록과 pending 예약을 합쳐 2건을 계산한다.
- 기존 초과 pending은 삭제되거나 실패 횟수가 증가하지 않고 다음 KST 날짜까지 보류된다.
- 내 원글 하나에 성공 기록과 예약을 합쳐 누적 2건까지만 대댓글이 허용된다.
- 내 원글 제한은 날짜가 바뀌어도 초기화되지 않는다.
- 일반 댓글 성공 5건 뒤 처리 가능한 대댓글 정확히 1건이 우선된다.
- 대댓글 후보가 없거나 실패해도 일반 자동댓글 게시가 계속된다.
- 서버 재시작 후 5:1 카운터가 보존된다.
- 두 게시 경로가 동시에 writer를 사용하지 않는다.
- 기존 전체 테스트가 통과한다.
- 실행 중인 서버에서 상태 API로 제한과 카운터가 노출되고, 실제 로그에서 계정별 세 번째 일반 댓글이 게시되지 않는 것을 확인한다.
